# DramaCina

A web-based streaming aggregator for Asian short drama content, built with Laravel 13. The application aggregates data from multiple drama providers through a unified API and presents them in a clean, modern interface.

## Overview

DramaCina pulls content from seven different providers and displays them on a single platform, allowing users to browse, search, and watch short-form Asian dramas without switching between applications.

**Supported Providers**

- DramaBox
- ReelShort
- ShortMax
- NetShort
- Melolo
- FreeReels
- DramaNova

## Features

- Multi-provider content aggregation via parallel HTTP requests
- Hero banner carousel with auto-play and swipe support
- Per-provider indexed pages with tab-based filtering (trending, latest, theaters, etc.)
- Drama detail pages with synopsis, genre tags, and episode lists
- Episode player with subtitle support where available
- Watchlist management (authentication required)
- Response caching with stale-while-revalidate strategy (30-minute TTL)
- Fully responsive layout for desktop and mobile
- No external CSS framework dependency — custom vanilla CSS with CSS custom properties

## Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-username/dramacina-laravel.git
cd dramacina-laravel
```

**2. Install PHP dependencies**

```bash
composer install
```

**3. Install Node dependencies and compile assets**

```bash
npm install
npm run build
```

**4. Configure the environment**

```bash
cp .env.example .env
php artisan key:generate
```

Edit `.env` as needed. The default configuration uses SQLite and the file cache driver, which requires no additional setup.

**5. Run database migrations**

```bash
php artisan migrate
```

**6. Start the development server**

```bash
php artisan serve
```

The application will be available at `http://127.0.0.1:8000`.

## Caching Strategy

The homepage fetches all seven providers in parallel using `Http::pool()`. Results are cached for 30 minutes. A stale copy is stored indefinitely and served as a fallback when the primary cache expires and a fresh fetch fails or is locked by another request. The cache key is versioned (`home_data_v4`) to allow forced invalidation after schema changes.

To clear all cached data:

```bash
php artisan cache:clear
```
